\chapter{因子预处理与基础暴露控制}
\label{ch:preprocessing}

\section{本章导读与学习目标}

第 \cref{ch:factor-construction} 已经为中期动量、账面市值比价值和 ROE 类质量生成统一的原始因子长表。本章以该长表为唯一因子输入，只处理冻结信号日股票池中
\texttt{is\_valid=True} 的 \texttt{raw\_value}，并且只连接信号日已经可得的历史行业、市值及相关元数据。目标是让不同量纲、不同分布的原始值变成可比较且可追溯的处理后因子，而不是借助处理步骤寻找更漂亮的结果。

完成本章后，读者应能够：

\begin{enumerate}
  \item 区分缺失、无效、极端但有效和普通有效四类状态；
  \item 在逐信号日、逐因子的横截面内计算覆盖率与分布统计；
  \item 使用配置化 MAD 缩尾、方向统一、排名或标准化；
  \item 测量因子的行业与市值暴露，并解释暴露不等于错误；
  \item 比较行业内处理与截面回归残差化的含义和限制；
  \item 保留原始值、步骤参数、处理状态与点时元数据；
  \item 用自动测试和匿名横截面手算证明没有跨日期或访问未来信息。
\end{enumerate}

\begin{principlebox}
[通用原则] 所有截面统计都只能来自同一
\texttt{signal\_date}、同一 \texttt{factor\_name} 的当期有效信号池。不得用全样本或未来日期的均值、标准差、分位数、行业映射或市值处理早期因子。
\end{principlebox}

本章不读取 \texttt{future\_return}、成交结果或组合表现，也不计算 IC、RankIC、分组收益或因子有效性结论。参数择优、多因子合成、组合与回测均留到后续章节。

\section{为什么预处理不是“数据美容”}

预处理的职责是让数值含义明确、尺度可比并暴露数据问题。它不能把不利观察隐藏掉，也不能把某种分布形状当作必须修正的缺陷。极端值可能来自录入错误，也可能是经济上真实但罕见的状态；行业差异可能是非目标暴露，也可能正是因子假设的一部分。只有先理解原因，才有资格选择处理方法。

因此，预处理方案应在查看后续收益前冻结。每一步都要能回答：使用了哪些当期样本，参数是多少，哪些记录被改变，哪些记录没有处理，以及处理后保留或删除了什么经济含义。若方法选择依据是最终回测表现，预处理就变成了隐蔽的参数搜索。

\begin{teachingbox}
[教学简化] 本章采用透明基线：
\texttt{raw\_value}
\(\rightarrow\) 有效样本筛选
\(\rightarrow\) 当日 MAD 缩尾
\(\rightarrow\) 固定方向统一
\(\rightarrow\) 当日标准化
\(\rightarrow\) 暴露诊断
\(\rightarrow\) 可选行业或市值处理
\(\rightarrow\) \texttt{processed\_value}。每一步均可由配置替换，但不得按后续表现挑选。
\end{teachingbox}

\section{缺失机制与覆盖率}

覆盖率的分母应是信号日冻结股票池，而不是“已经有值的记录数”。令
\(\mathcal{U}^{\mathrm{signal}}_t\) 为信号日股票池，令
\(\mathcal{V}_{f,t}\) 为因子 \(f\) 在 \(t\) 日满足有效条件且原始值有限的证券集合，则

\begin{equation}
  \mathrm{Coverage}_{f,t}
  =
  \frac{
    \left|\mathcal{V}_{f,t}\right|
  }{
    \left|\mathcal{U}^{\mathrm{signal}}_t\right|
  } .
  \label{eq:ch05-coverage}
\end{equation}

覆盖率至少要按因子和信号日报告，并进一步按历史行业与预先冻结的市值层检查。若某因子整体覆盖尚可，却长期缺失于某个行业或规模层，后续横截面可能代表的是一个被选择过的子样本。诊断还应拆分 \texttt{invalid\_reason}，避免把历史不足、财务尚不可得和非法分母混成同一种缺失。

贯穿案例的多因子阶段默认使用三个因子均有效的交集，因此还要报告交集覆盖率以及从单因子到交集的损失。行业中位数填补或缺失指示变量可以作为预注册敏感性方案，但不是本章默认基线，更不能因为某种填补后的表现较好才采用。

\section{为什么不能无条件填零}

零可能是合法的经济数值：方向统一后的动量为零、标准化值接近零，都有明确含义。把未知或无效填成零，会让“没有信息”伪装成“处于截面中间”，还会改变均值、标准差、排名、行业暴露和后续交集覆盖率。填零之后，原始缺失机制也很难恢复。

\begin{warningbox}
无效值保持缺失，信号记录保持存在。不得使用
\texttt{fillna(0)} 让记录进入缩尾、标准化或回归。若研究确需填补，必须把填补方法、适用分组、样本下限和缺失指示字段写入独立敏感性配置，并保留未填补基线。
\end{warningbox}

\section{截面处理必须逐信号日执行}

对每个因子 \(f\) 和信号日 \(t\)，先从冻结股票池取出当期记录，再选取
\texttt{is\_valid=True} 且 \texttt{raw\_value} 有限的集合
\(\mathcal{V}_{f,t}\)。中位数、MAD、分位数、均值、标准差、行业统计和回归设计矩阵都只能从这个集合构造。处理函数的基本分组键因此是
\texttt{["signal\_date", "factor\_name"]}。

跨日期共享统计量会把未来分布带到过去。即使全样本均值只由原始因子构成、没有显式收益标签，它仍包含未来日期信息，并会让不同时期的尺度相互污染。正确实现应为每个分组保存独立的样本数、边界、均值、标准差和处理状态。

\begin{practicebox}
最小防错断言包括：每个处理分组只含一个信号日和一个因子；统计样本均有效；处理前后记录总数一致；原始值逐元素不变；输入行业与市值的可得日不晚于信号日；输出按稳定主键确定性排序。
\end{practicebox}

\section{极端值识别与缩尾}

缩尾（winsorization）把超过边界的有效值压到边界，而不是删除记录。对某一
\((f,t)\) 截面，记有效原始值为 \(x_{i,t}\)，其中位数与绝对中位差（median absolute deviation, MAD）定义为

\begin{align}
  m_t
  &=
  \operatorname{median}_{i\in\mathcal{V}_{f,t}}
  \left(x_{i,t}\right),
  \label{eq:ch05-median}\\
  \mathrm{MAD}_t
  &=
  \operatorname{median}_{i\in\mathcal{V}_{f,t}}
  \left|x_{i,t}-m_t\right|.
  \label{eq:ch05-mad}
\end{align}

设 \(k\) 为边界倍数、\(c\) 为尺度常数，则教学边界写为

\begin{equation}
  \ell_t=m_t-kc\,\mathrm{MAD}_t,
  \qquad
  u_t=m_t+kc\,\mathrm{MAD}_t,
  \label{eq:ch05-mad-bounds}
\end{equation}

并令
\(x^{\mathrm{clip}}_{i,t}=\min\{\max(x_{i,t},\ell_t),u_t\}\)。
\(k\)、\(c\)、最小样本数和备用规则都属于配置，而非行业常数。当 MAD 为零、有效样本过少或截面退化时，应输出
\texttt{MAD\_DEGENERATE} 或
\texttt{INSUFFICIENT\_CROSS\_SECTION}，再按冻结配置选择“不缩尾”或其他降级规则。

分位数缩尾是可解释的替代方案，但它同样必须逐日计算，并预先固定上下分位点、插值方式和小样本规则。不能通过最终因子表现选择 MAD 或分位数方案。

\section{排名、标准化和方向统一}

方向统一来自第 4 章定义卡。令 \(d_f\in\{-1,+1\}\) 为冻结的
\texttt{direction\_sign}，先得到
\(x^{\mathrm{dir}}_{i,t}=d_f x^{\mathrm{clip}}_{i,t}\)。这个符号不能根据后续收益翻转。

逐信号日 z-score 定义为

\begin{equation}
  z_{i,t}
  =
  \frac{
    x^{\mathrm{dir}}_{i,t}-\mu_t
  }{
    \sigma_t
  },
  \label{eq:ch05-zscore}
\end{equation}

其中 \(\mu_t\) 与 \(\sigma_t\) 只由当日有效截面计算。若
\(\sigma_t\) 为零或低于配置阈值，程序必须停止标准化或进入明确降级状态，不能强行除法。

排名只保留相对次序，对尾部距离不敏感；z-score 保留相对距离，但更依赖分布和尺度。一种双边归一化排名可写为

\begin{equation}
  r_{i,t}
  =
  2\frac{\operatorname{rank}_{i,t}-1}{n_t-1}-1,
  \qquad n_t>1.
  \label{eq:ch05-rank}
\end{equation}

并列值规则、\(n_t\) 下限、排名区间和标准差自由度都要由配置固定。\cref{tab:ch05-method-comparison} 比较四种常用操作。

\section{行业与市值暴露}

因子与行业或规模相关并不自动意味着实现错误。暴露诊断首先描述“处理前后与什么共同变化”，然后才讨论是否需要控制。市值诊断可在每个信号日计算因子与
\(\log(\mathrm{MarketCap}_{i,t})\) 的截面相关，或在截面回归中记录对应系数；行业诊断则比较各历史行业的因子均值、中位数和覆盖率。

行业标签和市值必须在 \(t\) 时点已经生效或可得。用今天的行业分类回填历史，或用信号日之后市值修复缺失，都会重新引入第 2 章已经禁止的未来信息。市值非正、单位异常、行业缺失或生效区间不覆盖 \(t\) 时，应保留明确状态。

\begin{confirmbox}
[需实际确认] 行业分类体系及层级、市值采用何种口径、对数变换前的合法性规则、市值层划分、行业变更生效时间和历史版本保存方式均依赖数据源与团队规范。本章只规定点时边界，不虚构字段映射。
\end{confirmbox}

\section{行业内处理}

行业内处理先按同一信号日的历史行业分组，再在每个行业内部执行排名、去均值或标准化。它可以减少行业均值差异，并让证券更多地与同业比较。若采用行业内去均值，可写为

\begin{equation}
  z^{\mathrm{within}}_{i,t}
  =
  z_{i,t}
  -
  \overline{z}_{g(i,t),t},
  \label{eq:ch05-within-industry}
\end{equation}

其中 \(g(i,t)\) 是证券 \(i\) 在 \(t\) 时点的历史行业，
\(\overline{z}_{g,t}\) 是当日该行业有效证券均值。

行业内处理并不控制行业内部的市值梯度，也可能让样本很小的行业产生不稳定分数。缺失行业、小行业和行业内标准差过小必须触发冻结降级规则，例如保留全市场标准化值、只去行业均值或标记不可处理；具体选择不能临时决定。

\section{截面回归残差化}

截面回归残差化把方向统一后的标准化因子作为被解释变量，用当期点时市值和行业哑变量解释其横截面差异。对每个信号日独立估计

\begin{equation}
  z_{i,t}
  =
  \alpha_t
  +
  \beta_t\log\!\left(\mathrm{MarketCap}_{i,t}\right)
  +
  \sum_{g=1}^{G-1}
  \gamma_{g,t}\,
  \mathrm{Industry}_{i,g,t}
  +
  \varepsilon_{i,t},
  \label{eq:ch05-residualization}
\end{equation}

并可将残差 \(\varepsilon_{i,t}\) 作为一种
\texttt{processed\_value}。省略一个行业作为基准只是避免完全共线的一种实现方式；截距、哑变量编码和缺失行业规则都必须固定。

回归仍以同一 \((f,t)\) 截面为单位。样本数必须明显大于有效参数数；设计矩阵共线、秩不足、小行业或市值缺失时，应明确失败或降级。是否使用加权回归、权重如何定义、残差是否再标准化，均属于研究口径而不是默认事实。

\begin{confirmbox}
[需实际确认] 等权或加权回归、行业哑变量编码、缺失行业类别、最小样本数、秩判断容差、异常市值处理和残差再标准化规则，都应在配置中冻结并由人工核验。
\end{confirmbox}

\section{中性化可能删除什么信息}

中性化不是机械必选步骤。账面市值比的分母直接含市值，行业资产结构也可能是价值假设的一部分；ROE 的行业差异可能反映商业模式；动量可能与规模、波动或流动性共同变化。残差化会删除能够被所选暴露解释的线性部分，其中既可能有非目标风险，也可能有原假设希望保留的信息。

因此，处理前后必须比较覆盖率、分布、方向一致性、原版本与处理版本的截面相关、行业均值差异以及与对数市值的关系。暴露下降只说明数学关联被削弱，不代表因子“更好”，更不能替代第 6 章的有效性与可交易性检验。

\section{三个教学因子的处理选择}

三个因子共享“有效筛选—当日缩尾—方向统一—当日标准化”的主基线，但暴露处理不必相同：

\begin{itemize}
  \item \textbf{中期动量：}检查价格跳跃和极端原始值，保留原始动量，再做逐日缩尾与标准化；同时诊断行业、市值、波动和流动性暴露。是否中性化取决于研究问题。
  \item \textbf{账面市值比：}只处理第 4 章已判定有效的正值。对数变换可作为配置化替代方案，但不是必选步骤。行业资产结构差异与分母中的市值都要求重点检查；原始、行业内和行业加市值残差版本应并列保留。
  \item \textbf{ROE 类质量：}检查小分母导致的长尾，但不回头修改第 4 章有效性定义。比较原始、缩尾和标准化版本，并诊断行业、杠杆与规模相关性；不了解经济含义时不得机械去除全部暴露。
\end{itemize}

\section{pandas 风格最小实现}

\cref{lst:ch05-preprocess} 展示逐信号日、逐因子的基线框架。代码使用概念字段，残差化由独立小函数表示，不引入复杂优化库。

\begin{lstlisting}[
  caption={逐信号日截面预处理的最小实现},
  label={lst:ch05-preprocess}
]
FORBIDDEN = {
    "future_return", "execution_status",
    "portfolio_weight", "backtest_result",
}
GROUP = ["signal_date", "factor_name"]
ORDER = GROUP + ["security_id"]

def preprocess_cross_section(frame, config):
    assert FORBIDDEN.isdisjoint(frame.columns)
    assert frame["direction_frozen"].all()
    market_known = frame["market_cap_available_date"].notna()
    assert (
        frame.loc[market_known, "market_cap_available_date"]
        <= frame.loc[market_known, "signal_date"]
    ).all()
    industry_known = frame["industry_asof"].notna()
    assert (
        frame.loc[industry_known, "industry_start"].le(
            frame.loc[industry_known, "signal_date"]
        )
        & frame.loc[industry_known, "signal_date"].lt(
            frame.loc[industry_known, "industry_end"]
        )
    ).all()

    out = frame.sort_values(ORDER).copy()
    raw_before = out["raw_value"].reset_index(drop=True).copy()
    pieces = []

    for _, cross_section in out.groupby(GROUP, sort=True):
        cs = cross_section.copy()
        valid = cs["is_valid"] & cs["raw_value"].notna()
        cs["clipped_value"] = None
        cs["standardized_value"] = None
        cs["processed_value"] = None
        cs["processing_status"] = "INPUT_INVALID"

        x = cs.loc[valid, "raw_value"]
        if len(x) < config["min_cross_section"]:
            cs.loc[valid, "processing_status"] = \
                "INSUFFICIENT_CROSS_SECTION"
            pieces.append(cs)
            continue

        median = x.median()
        mad = (x - median).abs().median()
        if mad <= config["mad_floor"]:
            cs.loc[valid, "processing_status"] = "MAD_DEGENERATE"
            pieces.append(cs)
            continue

        radius = config["k"] * config["mad_scale"] * mad
        clipped = x.clip(median - radius, median + radius)
        directed = clipped * cs.loc[valid, "direction_sign"]
        std = directed.std(ddof=config["ddof"])
        if std <= config["std_floor"]:
            cs.loc[valid, "processing_status"] = "STD_DEGENERATE"
            pieces.append(cs)
            continue

        standardized = (directed - directed.mean()) / std
        cs.loc[valid, "clipped_value"] = clipped
        cs.loc[valid, "standardized_value"] = standardized
        cs.loc[valid, "processed_value"] = standardized
        cs.loc[valid, "processing_status"] = "OK"

        if config["exposure_method"] == "residual":
            residual, status = residualize_asof(
                standardized,
                cs.loc[valid, "industry_asof"],
                cs.loc[valid, "log_market_cap_asof"],
                config,
            )
            cs.loc[valid, "processed_value"] = residual
            cs.loc[valid, "processing_status"] = status
        pieces.append(cs)

    result = (
        concat(pieces)
        .sort_values(ORDER)
        .reset_index(drop=True)
    )
    assert result["raw_value"].equals(raw_before)
    assert len(result) == len(frame)
    result["preprocess_method"] = config["method_name"]
    result["preprocess_version"] = config["version"]
    result["exposure_method"] = config["exposure_method"]
    return result
\end{lstlisting}

示例中的 \texttt{residualize\_asof} 必须在单个分组内构造截距、一个基准行业之外的行业哑变量和点时对数市值，并在拟合前检查样本数、缺失、矩阵秩与共线性。若检查失败，它应返回明确状态而不是悄悄退回另一模型。若配置选择排名，应在相同位置替换 z-score 分支，并固定并列规则。

\section{自动测试与人工核对}

自动测试至少覆盖以下边界：

\begin{enumerate}
  \item 使用全样本均值或标准差的实现被识别为错误；
  \item 不同信号日不共享均值、分位数、MAD 或缩尾边界；
  \item \texttt{is\_valid=False} 的原始值不进入任何截面统计；
  \item 缺失值不被自动填成零；
  \item 处理后 \texttt{raw\_value} 与输入逐元素完全一致；
  \item MAD 或标准差为零时不产生无穷值，并输出退化状态；
  \item 有效样本过少时输出
  \texttt{INSUFFICIENT\_CROSS\_SECTION}；
  \item 历史行业生效区间覆盖 \texttt{signal\_date}；
  \item 市值可得日不晚于 \texttt{signal\_date}；
  \item 小行业或缺失行业触发冻结降级规则；
  \item 回归设计矩阵共线或秩不足时明确失败或降级；
  \item \texttt{direction\_sign} 只能来自冻结定义卡；
  \item 处理函数无法访问收益标签、成交结果或组合表现；
  \item 相同输入和配置生成字节级或数值容差内一致结果；
  \item 处理前后记录数、有效数、覆盖率和缺失原因可核对。
\end{enumerate}

\begin{practicebox}
人工穿行至少选取两个信号日、三个因子、两个行业和不同市值层；每个截面核对有效集合、中位数、MAD、边界、方向符号、均值、标准差、行业统计与市值关系。另选一个缺失值、一个极端有效值、一个小行业和一个秩不足案例，记录预期状态、程序状态、参数版本与核验人备注。
\end{practicebox}

\section{常见错误}

\begin{warningbox}
常见错误包括：在全样本上缩尾或标准化；先填零再计算统计量；让无效原始值参与分位数；覆盖
\texttt{raw\_value}；跨信号日复用边界；从未来行业或市值表取值；按后续表现翻转方向、选择阈值或填补方法；把行业内处理和残差化混为一谈；设计矩阵秩不足仍返回数值；以及把暴露下降直接表述为因子改善。
\end{warningbox}

这些错误往往不会让程序崩溃，却会改变覆盖、分布和因子经济含义。发现问题后应从分组键、有效集合与点时输入重新核对，而不是查看后续表现后反推一个“更合适”的参数。

\section{与第 6 章《单因子检验与可交易性诊断》的衔接}

本章交付的是带覆盖率、分布参数、处理状态和暴露说明的可追溯因子版本。第 6 章可以在预先冻结的版本上评价预测力、单调性、换手与可交易性，但不得在看到检验结果后回到本章反复调整缩尾阈值、填补方法、方向或中性化方案。

进入下一章前，应同时冻结原始版本、主基线版本和少量预注册敏感性版本，并写清
\texttt{processed\_value} 的准确含义。这样，后续差异才能解释为方法选择的影响，而不是无法追溯的处理覆盖。
