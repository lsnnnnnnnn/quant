\chapter{数据与时间边界}
\label{ch:data-timing}

\section{本章导读}

量化研究面对的第一道问题，不是某个因子怎样计算，而是：\keyterm{在某个历史研究时点，研究者实际能够看到哪些信息？} 同一个数值可以同时关联报告期、公告日、入库时间和信号日；若只保留数值而丢失这些时间语义，未来信息就可能被悄悄搬到过去。程序仍能运行，结果甚至会显得更整齐，但研究问题已经被改写。

本章围绕“信息何时可用”建立最小、可核对的时间框架。我们先区分关键日期，再用点时数据（Point-in-Time Data）定义可见信息集；随后通过一个极小财务数据例子比较错误连接与正确连接，并把同样的原则应用到历史指数成分、行业分类、交易状态和收益标签。重点不是记住供应商字段，而是学会为每一次匹配留下可审计的时间证据。

\begin{casebox}
\textbf{贯穿案例：宽基股票池三因子周频约束多头策略。} 每周最后一个交易日收盘后冻结信息，计算中期动量、账面市值比价值和 ROE 类质量信号，再形成综合分与目标权重。下一交易日才按明确、可配置的日频价格代理模拟成交。本章只定义信息边界和事件顺序，不产生任何策略业绩。
\end{casebox}

\section{学习目标}

完成本章后，读者应能够：

\begin{enumerate}
  \item 区分自然日、交易日、报告期、公告日、可得日、信号日、成交日、持有期和标签窗口；
  \item 用点时条件判断一条记录能否进入某个历史信号截面；
  \item 识别财务信息提前、当前成分回填和最新修订值回填造成的偏差；
  \item 解释普通等值连接与 as-of 连接的适用边界；
  \item 画出信号、目标权重、订单、成交、持有和收益标签的事件顺序；
  \item 为关键数据建立最小可得性规则，并完成自动检查和人工穿行核对；
  \item 在历史修订或精确时间戳无法恢复时，清楚披露限制而不是假装点时完备。
\end{enumerate}

\section{为什么时间边界是量化研究的第一道质量闸门}

横截面研究把许多证券在同一信号日排到一起。这个“同一日”只对计算截面成立，并不意味着各字段在那一天都已发布。例如，价格可能在收盘后可知，财务数值对应更早的报告期，却在更晚的公告日才公开；指数成分和行业分类还各有生效区间。若研究底表只按证券和报告期合并，数据表面上更完整，实质上却把尚未公开的信息用于过去的排序。

这种错误会系统性美化研究：较晚公布的记录被提前使用，历史覆盖率虚高；只保留今天仍存在的证券或当前成分，会删去过去失败或退出的样本；用下一交易日是否可成交反向删除信号日证券，则把成交阶段才知道的状态泄漏到信号阶段。它们共同的特征是\riskterm{用后来知道的事实重写当时的信息集}。

\begin{principlebox}
\textbf{[通用原则]} 每一个信号值都必须能回答三个问题：它描述什么时点的经济状态；它何时首次对研究者可得；它为何有资格进入该信号日的信息集。无法回答时，先停止因子计算，不能用“数据库里已有该列”代替时间证据。
\end{principlebox}

\section{Point-in-Time 数据的正式定义}

点时数据不是“带日期的数据”，而是能够重建历史时点可见状态的数据。设证券 $i$ 的第 $k$ 条历史记录为 $x_{i,k}$，其最早可得时点为 $a_{i,k}$。在信号截止时点 $t$，可使用的最近记录定义为

\begin{equation}
  x_i^{\mathrm{PIT}}(t)=x_{i,k^\star(i,t)},\qquad
  k^\star(i,t)=\operatorname*{arg\,max}_{k:\,a_{i,k}\le t} a_{i,k}.
  \label{eq:ch02-pit-definition}
\end{equation}

\cref{eq:ch02-pit-definition} 包含两个约束。第一，候选记录的可得时点必须不晚于信号截止；第二，只能在同一证券内部选择最近记录。若集合为空，正确结果是缺失，而不是向未来寻找一条“最近”的记录。若同一可得时点存在多个修订版本，还需要稳定的版本顺序或数据冻结规则。

点时正确性还要求可追溯：匹配结果至少保留证券标识、原始记录标识、报告期、公告时点、可得时点、版本和信号时点。只输出最终因子值而丢弃匹配来源，会使后续抽查无法判断它是否真的满足点时条件。

\section{财务数据中的报告期、公告日和可得日}

报告期回答“这项财务信息描述哪个期间”，公告日回答“它何时对外披露”，可得日回答“研究流程何时能够稳定使用”。三者可能不同。某项指标描述 20X1 年 3 月末的状态，并不意味着 3 月末即可用于信号；如果 4 月下旬才披露，那么此前的任何信号日都不能看到该值。

公告日也未必足够精细。若数据只有日期而没有时刻，就无法仅凭日期判断信息是在收盘前还是收盘后出现；若数据供应链次日才完成入库，研究可得时点还会晚于公开披露。此时应采用经过核验的保守规则，并记录规则对覆盖率的影响。

\begin{confirmbox}
\textbf{[需实际确认]} 需要核验公告时间戳粒度、时区、入库延迟、历史修订保存方式和团队信号截止。\textbf{[机构实现]} 生产数据表、字段映射、批处理完成时间与质量冻结流程由团队内部平台决定，本讲义不假设其名称或确定口径。
\end{confirmbox}

\section{普通等值 join 与 as-of join}

普通等值连接要求键完全相同，适合证券主键映射或已经构造好的同日快照。财务公告是不规则事件，信号日通常不会等于可得日，因此仅按日期等值连接会产生大量缺失；若随后随意向前或向后填充，又可能重新引入泄漏。as-of 连接直接表达“在该时点以前最近的一条记录”，但方向、分组和排序任何一项写错都会改变信息集。两者对比如\cref{tab:ch02-join-comparison}。

\section{首次披露、修订与历史版本}

同一报告期可能先首次披露，之后再修订。历史研究要问的是“当时能看到哪个版本”，而不是“今天数据库里哪个版本最完整”。若数据库只保留最新值，再把它回填到首次公告日，就会让修订后的信息提前出现。

稳妥的数据模型应把报告期、版本标识、公告时点、可得时点和替代关系分开保存。对信号日 $t$，先筛选 $a_{i,k}\le t$ 的版本，再按可得时点与版本顺序选择当时有效记录。人工抽查时，应能从匹配值追溯到原始披露或修订记录。

如果无法恢复历史修订版本，不能声称数据点时完备。可以根据研究目的采取保守滞后、仅使用可验证的首次披露快照，或将相关字段排除；选择哪一种取决于数据证据和团队规范，必须记录对覆盖率与结论边界的影响。

\section{历史指数成分与幸存者偏差}

用今天的指数成分回测过去，相当于提前知道哪些证券后来仍被保留。退出、退市或表现不佳的证券可能从历史样本中消失，研究对象因此不再是当时真实可选的股票池。这是幸存者偏差的一种常见来源。

正确做法是保存历史成分的生效起止区间，并在信号日查询当时有效的记录。成分调整的公告日与生效日承担不同作用：是否纳入信号日股票池，应遵循已确认的指数历史口径和生效规则；不能看到下一交易日的实际状态后再回头改写信号日成分。还应保留稳定证券标识，避免代码变更或退市证券在历史连接中丢失。

\begin{principlebox}
\textbf{[通用原则]} 股票池成员资格只能由信号截止及以前可得、且在该时点有效的记录决定。当前成分列表不能回填历史；未来退出与下一交易日不可成交也不能成为删除过去样本的理由。
\end{principlebox}

\section{历史行业分类与生效时间}

行业分类同样会变化。若把当前行业标签复制到全部历史，过去的行业中性化、行业约束和分组比较都会使用未来分类。正确结构应至少包含证券、分类标准、行业代码、有效起点和有效终点，并明确区间端点是闭区间还是半开区间。

对信号日的匹配应满足“有效起点不晚于信号日，且有效终点晚于信号日或为空”。同一证券、同一分类标准的区间不应重叠；未知终点不能被随意解释为永久有效。分类公告与生效之间如何处理，以及历史分类是否被供应商回溯修订，属于需实际确认的口径。

\section{Python/pandas 风格最小示例}

\cref{lst:ch02-merge-asof} 只展示时间连接的核心。两侧先按各自时间键升序排序，\texttt{by="security\_id"} 限制在同一证券内部匹配，\texttt{direction="backward"} 表示只寻找可得日不晚于信号日的最近记录。pandas 官方接口要求两侧按合并时间键升序，并允许先按 \texttt{by} 键匹配。\footnote{pandas.merge\_asof 官方文档：\url{https://pandas.pydata.org/docs/reference/api/pandas.merge_asof.html}。实际项目应固定并记录 pandas 版本。}

\begin{lstlisting}[caption={按证券进行向后点时匹配的最小示例},label={lst:ch02-merge-asof}]
import pandas as pd

signals = signals.sort_values(["signal_date", "security_id"])
facts = facts.sort_values(["available_date", "security_id"])

matched = pd.merge_asof(
    signals,
    facts,
    by="security_id",
    left_on="signal_date",
    right_on="available_date",
    direction="backward",
    allow_exact_matches=True,
)

valid_time = matched["available_date"].isna() | (
    matched["available_date"] <= matched["signal_date"]
)
assert valid_time.all()

pre_announce = matched["signal_date"] < matched["announce_date"]
assert matched.loc[pre_announce, "value"].isna().all()
\end{lstlisting}

断言不是装饰。若出现可得日晚于信号日，说明方向、排序或时间字段至少有一项错误；若不同证券之间发生匹配，说明分组键缺失或主键映射有问题。真实项目还应检查时区、缺失值、重复版本和可接受的最大滞后，但不应把未知口径藏在代码默认值里。

\section{自动化数据质量检查}

自动检查应在生成因子前执行，并输出失败样本而不只是一个布尔值。检查结果还要区分“数据缺失”“规则未知”和“确定违规”，避免把所有问题都用删除行处理。

\begin{practicebox}
\textbf{正确实践：最小自动化检查清单}
\begin{enumerate}
  \item 检查预期主键唯一，例如信号表的证券与信号日、财务表的证券、报告期、版本和可得时点组合；
  \item 检查日期范围与先后关系，识别异常未来日期和不可能的报告期；
  \item 对已匹配记录断言公告日不晚于信号截止；
  \item 断言 \texttt{available\_date} 不晚于 \texttt{signal\_date}，并输出违反记录；
  \item 比较历史成分来源与回测截面，禁止当前成分整段回填；
  \item 检查同一证券、同一行业标准的生效区间不得重叠；
  \item 按信号日、数据类别和股票池统计点时快照覆盖率，并解释突变；
  \item 检查公告、可得、生效或成交字段中的异常未来日期；
  \item 识别同一报告期的重复修订记录，并验证版本顺序；
  \item 汇总未知或缺失生效区间，禁止默认解释为永久有效。
\end{enumerate}
\end{practicebox}

覆盖率本身不是越高越好。若修复点时逻辑后覆盖率下降，可能只是过去错误匹配被移除；应检查下降发生在哪些日期和证券，而不是为了恢复覆盖率重新引入未来值。

\section{常见错误}

\begin{warningbox}
常见错误并不只发生在财务连接。以下行为都在用未来事实重写过去：
\begin{enumerate}
  \item 把报告期末当成财务信息可得日，或按报告期直接前填；
  \item 使用 forward 或 nearest 时间连接，却没有证明匹配记录在信号截止前可得；
  \item 用全样本最新修订值替代历史首次披露或当时版本；
  \item 用当前指数成分、当前行业或现存证券列表回填历史；
  \item 用下一交易日停牌或受限状态删除信号日样本；
  \item 用信号日收盘信息计算信号，又假设无条件以同一收盘价成交；
  \item 只检查最终行数，不保留匹配来源、时间间隔和未匹配原因；
  \item 将缺失一律删除，使未知信息在样本选择中形成新的偏差。
\end{enumerate}
\end{warningbox}

发现错误后，应先定位它改变了哪些日期、证券和字段，再重新生成底表并比较差异。不要只在最终指标层面做补丁，因为被污染的信息集可能已经影响股票池、因子预处理、组合和标签。

\section{与第 3 章《股票池、可交易边界与收益标签》的衔接}

本章建立了“何时可见”的底层约束，但尚未决定哪些证券应进入研究股票池、哪些状态只影响成交，以及未来收益标签如何精确定义。第 3 章将在点时信息集之上区分研究池、信号日股票池和成交日可交易状态，设计纳入与剔除原因，并把标签起止时点落实为可测试的边界规则。
