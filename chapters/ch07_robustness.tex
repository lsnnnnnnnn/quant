\chapter{稳健性、样本外与反过拟合}
\label{ch:robustness}

\section{本章导读与学习目标}

第 \cref{ch:single-factor} 已经在冻结股票池、标签和因子版本上建立了覆盖率、预测力、暴露、换手与成本五维证据。本章不再寻找新的因子公式，而是追问：\keyterm{这些证据是否依赖某段样本、某个参数或研究者的选择？} 一个因子可以在全样本平均意义上表现整齐，却只在少数年份、单一股票池或某个孤立参数点成立；若研究者反复查看测试结果并据此改规则，“样本外”也会逐渐变成另一段样本内。

稳健性检验的目标不是证明因子在任何条件下都有效，而是识别证据能够被复现的区域、明确适用边界，并诚实保留失败和证据不足。完成本章后，读者应能够：

\begin{enumerate}
  \item 按时间顺序区分训练、验证、测试和最终留出区间的职责；
  \item 解释时间序列为何不能随机打乱后使用普通交叉验证；
  \item 比较扩展窗口与滚动窗口的适用问题和局限；
  \item 围绕冻结主配置设计少量预注册参数扰动；
  \item 检查时间、行业、市值、流动性、股票池与事前市场状态子样本；
  \item 识别参数平坦区域与单点尖峰之间的证据差异；
  \item 用实验台账记录所有尝试、失败状态和测试集访问次数；
  \item 汇总稳健性矩阵并给出冻结、带条件进入、返工、停止或证据不足的决定。
\end{enumerate}

\begin{principlebox}
\textbf{[通用原则]} 看过测试结果后再修改参数，测试集就已经承担了验证集职责；最终留出集只有在研究设计完全锁定后才允许查看一次。任何重复查看都必须记录，不能通过更换名称恢复“未使用”状态。
\end{principlebox}

本章不进行新公式搜索、多因子合成、组合构建、正式策略回测、复杂机器学习交叉验证或大规模自动因子挖掘。进入第 8 章前，必须冻结最终因子版本；不能再根据第 8—10 章的组合结果回头挑选本章版本。

\section{为什么稳定性比单次好结果更重要}

单次好结果只能说明“在某个已经观察到的设置里出现了有利证据”，不能说明证据对轻微口径变化、时间推进或样本替换具有抵抗力。若一个窗口参数恰好产生较高 RankIC，而相邻合理参数全部退化，更可能反映噪声、数据边界或选择偏差，而不是清楚的经济机制。

稳健性也不等于每个子样本必须同样优秀。行业结构、信息覆盖和市场环境会变化，因子可能存在合理适用边界。关键是边界能否由事前假设解释，失败区域是否完整披露，以及进入下一阶段的决定是否由冻结规则而非事后情绪产生。

\begin{warningbox}
只展示最佳参数、最佳起止日期和最有利股票池，会把研究者的搜索过程隐藏在最终表格之外。即使每个单独计算都没有代码错误，选择性汇报仍会系统性夸大证据。
\end{warningbox}

\section{为什么时间序列不能随机打乱}

普通随机划分会把较晚时期的记录放入训练区间，又把较早记录放入验证或测试区间。即使因子值本身满足点时要求，模型选择过程仍然看到了未来市场结构、覆盖率和分布信息。周度标签还可能跨越划分边界；若相邻信号的标签窗口重叠，随机打乱会进一步放大信息依赖。

正确做法是按信号日排序后切分连续区间，并检查特征截止、标签起止和分割边界。若标签窗口跨越下一段起点，应按冻结规则设置间隔、归属或剔除状态，不能让同一未来区间同时支持参数选择和测试。随机种子不能修复时间语义错误。

\begin{practicebox}
\textbf{[正确实践]} 对每个区间断言信号日单调递增；前一区间结束早于后一区间开始；任何用于选择的标签终点不越过允许边界；原始数据顺序、切分后索引和标签窗口均可追溯。
\end{practicebox}

\section{扩展窗口与滚动窗口}

扩展窗口从固定历史起点逐步增加新数据，观察累积证据是否随样本增加而保持方向。滚动窗口则只保留最近固定长度历史，更敏感于衰减和结构变化。设按时间排序的信号日为 \(t_1,\ldots,t_K\)，扩展窗口和长度为 \(W\) 的滚动窗口可分别写为

\begin{align}
  \mathcal{T}^{\mathrm{expand}}_k
  &= \{t_1,\ldots,t_k\},
  \label{eq:ch07-expanding-window}\\
  \mathcal{T}^{\mathrm{roll}}_{k,W}
  &= \{t_{k-W+1},\ldots,t_k\},\qquad k\ge W.
  \label{eq:ch07-rolling-window}
\end{align}

\section{参数敏感性与尖峰风险}

\section{时间子样本与滚动指标}

时间子样本应在看结果前按研究问题定义，例如固定日历段、数据制度变化前后或预先登记的阶段。每段继续使用第 6 章相同的覆盖率、RankIC、分组单调性、暴露、换手和成本敏感性口径，避免因子在不同区间接受不同评分规则。

滚动指标用于观察证据演变，但窗口之间高度重叠，不能把每个滚动点当作独立试验。报告应同时展示窗口有效期数、标签覆盖和失败状态；短窗口的剧烈波动可能只是估计不稳定。若衰减只在覆盖率断点附近出现，应先核查数据和股票池版本，再讨论经济失效。

\section{行业、市值和股票池子样本}

行业子样本回答信号是否由少数行业驱动；市值和流动性子样本回答证据是否只存在于某一交易特征区间；股票池替代则检验结论对基础池定义的依赖。所有子样本都必须使用信号日点时分类与预注册边界，并保留完整分母。

子样本覆盖不足时应返回 \texttt{INSUFFICIENT\_SUBSAMPLE}，而不是计算伪精确指标。股票池替代不意味着可以使用当前成分回填历史；每个替代池都需要独立历史成分版本。某些行业或规模层不支持因子，并不自动否定全体证据，但必须进入适用边界和第 8 章版本说明。

\begin{confirmbox}
\textbf{[需实际确认]} 行业体系、市值与流动性分层、替代股票池、最小子样本数和历史成分版本均依赖实际数据。正文只冻结检查原则，不虚构具体阈值或指数名称。
\end{confirmbox}

\section{市场状态必须事前定义}

“牛市有效”“压力期失效”等表述只有在状态定义不依赖因子结果时才可检验。事后观察曲线再为有利或不利区间命名，会把结果本身用于构造解释变量。市场状态应使用事前可计算、点时可得且与因子表现无关的规则，并在实验台账中保存定义版本。

\begin{confirmbox}

\end{confirmbox}

\section{持有期和标签口径敏感性}

不同持有期回答不同预测问题，必须使用独立 \texttt{label\_version}，并重新核对信号截止、成交代理和标签端点。不能在同一标签列上改变名称，也不能因某个持有期表现更好就宣称原假设得到支持。

本章只允许少量预注册持有期或标签代理，用于判断第 3 章主标签结论是否过度依赖单一设置。比较时同时报告覆盖率、重叠程度、RankIC、分组单调性和换手含义。更短持有期可能提高换手和成本压力，更长持有期可能产生标签重叠；二者都需要在解释中披露。

\begin{confirmbox}
\textbf{[需实际确认]} 主持有期、替代持有期、成交价格代理、重叠标签处理及公司行动口径依赖真实研究协议。任何替代标签都要单独版本化，不能覆盖第 3 章主标签。
\end{confirmbox}

\section{多重尝试、数据窥探和选择性汇报}

每增加一个窗口、阈值、股票池或子样本，研究者就多了一次偶然发现有利结果的机会。若只保留最佳者，普通显著性直觉会过于乐观。即使本章不展开复杂多重检验方法，也必须记录假设家族、全部尝试和选择路径。

数据窥探不仅发生在直接查看收益时。反复观察 RankIC、分组单调性、覆盖率或成本代理并据此改配置，同样会消耗样本外信息。失败、报错和证据不足都应保留；测试集和最终留出集的每次查看也必须进入台账。

\begin{principlebox}
\textbf{[通用原则]} 研究记录的基本单位是“所有尝试”，不是“最终保留的尝试”。删除失败实验会让后来复核者无法评估选择次数，也会让团队重复踩坑。
\end{principlebox}

\section{一个匿名教学案例}

\begin{casebox}
\textbf{仅用于解释研究纪律和决策过程，不代表真实证券、真实数据库或实证研究结果。}

某匿名因子的主基线在训练区间与验证区间保持相同方向，覆盖状态也没有异常断点。预注册参数邻域包含主基线及少量相邻设置，其中一个参数点在验证区间异常突出，但其左右相邻点均不支持同样结论；另一个市值子样本因有效证券和有效期数不足，返回 \texttt{INSUFFICIENT\_SUBSAMPLE}。

研究者不能选择孤立尖峰并把它带入测试。若主基线满足事前最低要求，正确做法是保留主基线、完整记录尖峰与覆盖不足，并进入一次冻结测试；若主基线本身证据较弱且关键子样本无法评价，则正确结论是“证据不足”，而不是扩大搜索范围。
\end{casebox}

这个案例没有提供真实日期、RankIC、收益或阈值。它展示的核心是决策顺序：先冻结候选与规则，再运行验证；尖峰只触发风险标记，不获得自动选择权；覆盖不足保留状态；测试集在最终方案冻结后才可访问。

\section{Python 风格最小实现}

\cref{lst:ch07-robustness-suite} 展示稳健性套件的职责边界。代码假设因子和标签已经由前章冻结，只编排预注册实验，不重新计算公式或自动选择最佳参数。

\begin{lstlisting}[
  caption={稳健性检验与追加式实验台账的最小接口},
  label={lst:ch07-robustness-suite}
]
from hashlib import sha256

FORBIDDEN_INPUTS = {
    "combined_score", "portfolio_weight", "backtest_result"
}

def validate_time_splits(split_config):
    names = ["train", "validation", "test", "holdout"]
    intervals = [split_config[name] for name in names]
    for interval in intervals:
        assert interval["start"] < interval["end"]
    for left, right in zip(intervals, intervals[1:]):
        assert left["end"] < right["start"]
    assert split_config["ordered"] is True
    assert split_config["random_shuffle"] is False

def make_experiment_id(spec, code_commit, data_version):
    payload = canonical_json({
        "spec": spec,
        "code_commit": code_commit,
        "data_version": data_version,
    })
    return sha256(payload.encode()).hexdigest()[:16]

def run_robustness_suite(
    frozen_factor,
    frozen_labels,
    split_config,
    experiment_grid,
):
    assert FORBIDDEN_INPUTS.isdisjoint(frozen_factor.columns)
    assert frozen_factor["definition_frozen"].all()
    assert frozen_factor["direction_frozen"].all()
    assert frozen_factor["preprocess_frozen"].all()
    assert frozen_factor["signal_date"].is_monotonic_increasing
    assert frozen_labels["signal_date"].is_monotonic_increasing
    validate_time_splits(split_config)

    assert experiment_grid["registry_frozen"] is True
    assert experiment_grid["selection_splits"] == [
        "train", "validation"
    ]
    assert "test" not in experiment_grid["selection_splits"]
    assert "holdout" not in experiment_grid["selection_splits"]

    ledger_rows = []
    matrix_rows = []
    for spec in experiment_grid["experiments"]:
        experiment_id = make_experiment_id(
            spec,
            experiment_grid["code_commit"],
            experiment_grid["data_version"],
        )
        assert not ledger_store.exists(experiment_id)

        try:
            assert spec["registered"] is True
            assert spec["factor_version"] in \
                experiment_grid["allowed_factor_versions"]
            assert spec["parameter_changes"] in \
                experiment_grid["allowed_perturbations"]
            result = evaluate_frozen_spec_by_split(
                frozen_factor,
                frozen_labels,
                split_config,
                spec,
                expose_splits=spec["authorized_splits"],
            )
            status = result["status"]
            matrix_rows.extend(to_matrix_rows(result, spec))
        except Exception as error:
            status = "FAILED"
            result = {"error_type": type(error).__name__}
        finally:
            ledger_rows.append(make_ledger_row(
                experiment_id=experiment_id,
                spec=spec,
                status=status,
                result=result,
                test_access_count=audit.test_access_count,
                holdout_access_count=audit.holdout_access_count,
            ))

    ledger_store.append_only(ledger_rows)
    assert audit.holdout_access_count <= 1
    return {
        "ledger": ledger_rows,
        "robustness_matrix": matrix_rows,
        "frozen_candidates": freeze_by_registered_rules(
            matrix_rows, experiment_grid["decision_rules"]
        ),
    }  # never select the historical best parameter automatically
\end{lstlisting}

\texttt{evaluate\_frozen\_spec\_by\_split} 必须复用第 6 章评价口径；\texttt{freeze\_by\_registered\_rules} 只能执行事前决策规则，不能按历史最高指标排序。实际实现还需让最终留出访问由独立审计状态控制，并在失败时也成功追加台账记录。

\section{自动测试与人工核对}

自动测试至少覆盖：

\begin{enumerate}
  \item 样本区间顺序错误或任意重叠时失败；
  \item 输入信号日被随机打乱或不单调时失败；
  \item 测试集进入参数选择函数时失败；
  \item 最终留出集被重复访问时记录并失败；
  \item 未登记参数或配置不得运行；
  \item 因子、标签、股票池或预处理版本变化必须生成新实验编号；
  \item 既有 \texttt{experiment\_id} 不得被覆盖；
  \item 失败、报错和证据不足的运行必须进入台账；
  \item 事后市场状态标签不得作为预注册分组；
  \item 参数敏感性输出保留全部登记邻域，不只保留最佳点；
  \item 子样本覆盖或有效期不足时返回明确状态；
  \item 不同持有期必须使用独立标签版本；
  \item 相同输入、配置、代码和数据版本生成一致结果；
  \item 进入第 8 章的因子版本必须生成明确冻结记录；
  \item 本章代码不能访问多因子综合分、组合权重或正式回测结果。
\end{enumerate}

\begin{practicebox}
\textbf{[正确实践]} 人工核对至少选择一个正常实验、一个参数尖峰、一个覆盖不足子样本、一个失败运行和一次测试访问。逐项核对实验编号、事前理由、版本、时间边界、允许扰动、完整邻域、状态、结果位置、测试/留出访问次数和最终决定，并确认旧记录未被覆盖。
\end{practicebox}

还应抽取一个扩展窗口和一个滚动窗口，手工核对其信号日集合；抽取一个替代持有期，确认使用独立标签版本；抽取一个市场状态记录，确认定义时间早于结果生成时间。

\section{常见错误}

\begin{warningbox}
常见错误包括：随机打乱时间序列；让标签跨分割边界泄漏；把验证、测试和留出混为一类；看过测试后继续调参；反复查看最终留出；遍历大量窗口寻找最大值；只展示最佳参数；把孤立尖峰称为稳健；用事后行情命名市场状态；忽略子样本覆盖损失；让不同持有期共用标签版本；删除失败实验；覆盖旧台账；以及根据第 8—10 章组合结果返回挑选本章版本。
\end{warningbox}

这些错误的共同点是让研究者的选择权没有进入报告。修复时必须恢复完整实验历史、重新标记已经被查看的数据职责，并在必要时承认已经没有真正未使用的留出集。

\section{与第 8 章《因子冗余与简单多因子合成》的衔接}

本章结束时，每个候选因子只能携带一个明确冻结的公式、方向、预处理、股票池、标签和评价版本，以及已经披露的适用边界。第 8 章将判断不同因子是否提供增量信息，并使用透明方法形成简单综合信号；它不能重新打开本章参数搜索，也不能根据组合表现选择另一个历史版本。

若本章决定为“返回数据或实现修复”，应先完成新版本的全链条检验；若为“停止”或“证据不足”，不得为了凑齐三因子案例而强行纳入。只有通过或带条件通过的冻结版本，才有资格进入冗余分析和多因子合成。
